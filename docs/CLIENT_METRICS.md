**Client Metrics Guide**

부하 테스트와 운영 중 장애 분석 시 확인할 클라이언트 지표 정리.

클라이언트는 다른 클라이언트 상태를 알 필요가 없으므로, 해당 클라이언트 자신의 세션 상태만 기준으로 본다.

---

**읽는 순서**

`Connection -> Traffic -> Latency -> Protocol -> Runtime`

앞 단계에 이상이 있으면 뒤 단계 지표 해석이 왜곡될 수 있으므로 항상 앞 단계부터 확인.

---

**빠른 진단 흐름**

| Step | Group | 먼저 확인할 질문 |
|---|---|---|
| 1 | Connection | 접속과 연결 유지가 정상인가 |
| 2 | Traffic | 내가 보내고 받는 트래픽이 정상인가 |
| 3 | Latency | 응답이 늦어지고 있는가 |
| 4 | Protocol | 수신 데이터가 정상 형식인가 |
| 5 | Runtime | 클라이언트 자체가 병목인가 |

---

**1. Connection**

목적: 해당 클라이언트의 연결 상태 확인.

| Metric | 의미 | 이상 시 확인 |
|---|---|---|
| `connect latency (ms)` | 연결 시작부터 TCP 연결 완료까지 걸린 시간 | 서버 접근성 이상, 접속 지연 |
| `disconnect reason` | 연결 종료 원인 | `timeout`, `server close`, `recv fail`, `local close` 구분 |
| `session uptime (ms)` | 연결 1회당 유지 시간 | 세션 유지 불안정, 조기 종료 |

이 그룹에서 이상이 보이면 접속 불안정, 연결 유지 실패, 서버 접근성 이상 확인.

---

**2. Traffic**

목적: 클라이언트 기준 송수신 흐름 확인.

| Metric | 의미 | 이상 시 확인 |
|---|---|---|
| `send packets/sec` | 해당 클라이언트가 실제로 전송한 패킷 수 | 송신 부족, 송신 실패 |
| `recv packets/sec` | 해당 클라이언트가 실제로 수신한 패킷 수 | 수신 정체, 응답 감소 |
| `send bytes/sec` | 초당 송신 바이트 수 | 요청량 변화 |
| `recv bytes/sec` | 초당 수신 바이트 수 | 응답량 변화 |
| `send fail count` | 송신 실패 횟수 | 클라이언트 송신 경로 이상 |

이 그룹에서 이상이 보이면 송신 실패, 수신 정체, 클라이언트 기준 트래픽 이상 확인.

---

**3. Latency**

목적: 해당 클라이언트가 체감하는 응답 지연 확인.

| Metric | 의미 | 이상 시 확인 |
|---|---|---|
| `request RTT (ms)` | 요청 전송부터 대응 응답 수신까지 왕복 시간 | 서버 응답 지연, 네트워크 지연 |
| `response timeout count` | 응답 제한 시간 초과 횟수 | 응답 누락, 과도한 지연 |
| `time since last recv (ms)` | 마지막 패킷 수신 이후 경과 시간 | recv 정체, 유휴 상태, 수신 이상 |

이 그룹에서 이상이 보이면 서버 응답 지연, 네트워크 지연, 응답 누락 여부 확인.

---

**4. Protocol**

목적: 클라이언트가 받은 데이터의 형식 이상 확인.

| Metric | 의미 | 이상 시 확인 |
|---|---|---|
| `packet validation failure count` | 수신 패킷 검증 실패 횟수 | 프로토콜 불일치, 손상 데이터, 잘못된 직렬화 데이터 |

---

**5. Runtime**

목적: 클라이언트 프로세스 자체 상태 확인.

| Metric | 의미 | 이상 시 확인 |
|---|---|---|
| `client process CPU percent` | 클라이언트 프로세스 전체 CPU 사용률 | 클라이언트 자체 병목, 로컬 자원 문제 |
| `thread count` | 현재 스레드 수 | 의도하지 않은 스레드 증가, 누수 |

이 그룹에서 이상이 보이면 클라이언트 자체 자원 이상, 로컬 병목, 구조적 런타임 문제 확인.
